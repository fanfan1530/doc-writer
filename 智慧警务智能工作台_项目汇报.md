# 智慧警务智能工作台 v2.0 — 项目汇报材料

> 编制日期：2026年5月26日 | 版本：v2.0.0

---

## 一、项目概述

**智慧警务智能工作台**是一套面向基层公安民警的 AI 辅助执法办案系统。系统以"文书生成"为核心，融合大语言模型（LLM）、向量语义检索（ChromaDB）和法律法规知识库，实现从案情输入到法律文书定稿的半自动化流程，覆盖行政/刑事案件的文书编写、类案检索、案件分析和知识查阅等实战场景。

**核心价值**：将民警从繁琐重复的文书撰写中解放出来，降低法律引用错误率，统一文书格式规范，提升基层办案质效。

## 三、系统功能模块

### 3.1 工作台仪表盘（/dashboard）

首页概览面板，展示生成文书总数、七日内生成数、文书类型分布等统计卡片，支持一键跳转至历史文书和相关功能页面。

### 3.2 文书生成（/documents）—— 核心模块

**AI 智能生成模式**：民警输入案情描述（自然语言），系统自动完成以下流水线：
1. **要素抽取** — LLM 从案情中提取时间/地点/人员/案由等结构化字段
2. **事实润色** — 将口语化描述转为规范公安法律文书语言
3. **字段后处理** — 日期标准化、默认值填充、政府机关名称派生
4. **法条推荐** — ChromaDB 语义搜索 + 关键词兜底，推荐适用法条
5. **处罚依据兜底** — 智能匹配实体罚则条款，过滤程序性条款
6. **模板填充** — 将字段填入标准文书模板，生成完整文书

**手动填写模式**：按模板逐字段填写，适用于要素明确的简单案件。

**辅助功能**：
- 文件上传：上传 .doc/.docx 文书文件 → AI 提取并结构化案情摘要
- 案件时间线：输入案情 → LLM 提取时间序列事件 → 垂直时间线可视化
- 批量文书链：一次案情输入 → 同步生成多份关联文书（询问笔录 + 检查笔录 + 处罚决定书 + 告知笔录等）
- 法定期限检查：自动比对办案期限、告知期限、扣押期限

**支持文书类型（12种）**：

| 文书名称 | 适用场景 |
|----------|----------|
| 行政处罚决定书 | 行政案件最终处罚 |
| 检查笔录 | 人身/场所检查过程记录 |
| 现场勘查笔录 | 刑事/行政现场勘查 |
| 辨认笔录 | 照片/人员辨认程序 |
| 搜查笔录 | 刑事搜查过程记录 |
| 行政处罚告知笔录 | 处罚前告知程序 |
| 治安调解协议书 | 民间纠纷调解 |
| 扣押决定书及清单 | 涉案财物扣押 |
| 指认笔录 | 嫌疑人现场指认 |
| 结案报告 | 案件办结审批 |

### 3.3 AI 助手（/copilot）

独立全屏对话面板，支持 SSE 流式输出。内置 **8 个专业工具**：

| 工具 | 功能 |
|------|------|
| 法条检索 | 按案由/关键词检索适用法律法规 |
| 程序查询 | 查询办案程序规定和期限要求 |
| 文书生成 | 对话式文书生成 |
| 文书润色 | 对已有文书进行语言规范化 |
| 期限检查 | 检查案件各环节法定期限 |
| 案情分析 | 自动分析案件性质和定性要素 |
| 处罚参考 | 根据案情推荐处罚幅度 |
| 证据指引 | 列出该类案件的证据清单 |

支持**简洁/详细**双模式切换，适应不同场景需求。对话历史持久化存储，支持会话管理和上下文回溯。

### 3.4 类案检索（/cases）

民警输入案件描述，系统通过 ChromaDB 向量语义搜索匹配历史相似案例。支持按刑事/行政/民事类型筛选，展示案例相似度、关键事实、处理结果、适用法条。含约 20 个种子典型案例，覆盖盗窃、故意伤害、诈骗、交通肇事、寻衅滋事、赌博、吸毒等常见案由。

### 3.5 案件分析（/analysis）

两 Tab 设计：
- **要素时间线**：LLM 自动提取时间序列事件 → 垂直时间线可视化
- **案件定性分析**：AI 辅助分析案件性质、构成要件、定性要点

### 3.6 知识库（/knowledge）

分页浏览法律法规库，支持按关键词搜索和文书类型筛选。每张法条卡片展示法律名称、条款编号、全文内容、处罚幅度标签、适用文书类型标签，支持一键复制引用。当前收录《治安管理处罚法》《行政处罚法》《刑事诉讼法》《刑法》《反电信网络诈骗法》等核心法律约 36 条常用条款。

### 3.7 历史文书（/history）

浏览、搜索、筛选、查看、下载、复用历史生成的文书。支持按文书类型/关键词/时间范围筛选，详情抽屉展示完整文书内容和结构化字段，一键导出 Word 文档或回填参数到生成页面重新生成。

### 3.8 设置中心（/settings）

模型配置管理：添加/编辑/切换大模型提供商（OpenAI 兼容接口、Anthropic Claude），配置 API 地址、模型名称、API Key、温度参数、最大 Token 数、系统提示词。支持连接测试。

## 四、核心技术亮点

### 4.1 多层法条匹配保障

```
LLM 提取 penalty_basis
    ↓ 为空时触发兜底
FieldProcessor._fallback_law_search()
    ↓ 关键词匹配 + 程序性条款过滤 + 罚则加权
RAGRetriever.retrieve_laws()
    ↓ 向量语义搜索 + doc_type 预过滤
→ 最终输出处罚依据
```

通过三层保障（LLM 提取 → 关键词兜底 → RAG 向量搜索），确保法律引用准确率。对行政案件自动排除刑事法律，对程序性条款（传唤、询问时限、辨认、扣押等）降权过滤，优先匹配实体罚则。

### 4.2 字段后处理管线

统一的 `FieldProcessor` 处理 6 种文书类型，涵盖：日期标准化、政府机关派生、默认值智能填充、LLM 多余前缀清理、证据句去重、文书编号自动生成。

### 4.3 背景生成持久化

文书生成状态置于全局 Context，用户可在生成过程中自由切换页面（如查阅知识库、检索类案），生成完成后回到文书页直接查看结果。

### 4.4 生成质量监控

- 每次生成自动记录到 `generation_history` 表（输入/输出/耗时/模型/法条）
- 模板填充后检查未替换占位符，标记 `[待填写]` 提示
- 处罚依据为空时输出可见警告标记

## 五、系统部署

```
docker-compose up -d
```

三容器架构：
- **app** (port 8090)：FastAPI + React 静态文件
- **redis** (port 6379)：缓存与会话
- **nginx** (port 80)：反向代理

后端开发启动：
```bash
cd backend && uvicorn app.main:app --host 0.0.0.0 --port 8091 --reload
```

前端开发启动：
```bash
cd frontend && npm run dev  # port 5176
```
